# 支付宝支付功能说明

## 功能概述

本次更新为 springMall 电商系统添加了完整的支付宝支付功能，用户可以在订单页面选择支付宝进行支付。

## 实现内容

### 1. API 接口模块（`src/api/payment.js`）

新增了以下 API 接口：

- **`createAlipayPayment(orderNo)`** - 创建支付宝支付
  - 参数：订单号
  - 返回：支付单信息（包含支付表单 HTML）

- **`getPaymentStatus(paymentNo)`** - 查询支付状态
  - 参数：支付单号
  - 返回：支付状态信息

### 2. 支付页面更新（`src/views/user/Payment.vue`）

#### 功能特性

- **支持两种支付方式**：
  - 支付宝支付（推荐，已接入）
  - 模拟支付（测试用）

- **支付流程**：
  1. 显示订单信息（订单号、金额、收货信息）
  2. 选择支付方式
  3. 点击"确认支付"
  4. 自动跳转到支付宝收银台
  5. 支付完成后跳转回支付结果页

#### 技术实现

- 动态插入支付表单：后端返回的 HTML 表单会被自动插入到页面并提交
- 表单提交后在当前窗口跳转到支付宝收银台
- 支付完成后支付宝会重定向到 `/payment/result?paymentNo=xxx`

### 3. 支付结果页（`src/views/user/PaymentResult.vue`）

#### 功能特性

- **自动查询支付状态**：页面加载时自动查询支付结果
- **支付状态轮询**：如果支付状态为"待确认"，每 3 秒自动查询一次，最多 10 次
- **多种结果展示**：
  - ✅ **支付成功**：显示订单号、金额、交易号等信息，提供"查看订单"按钮
  - ❌ **支付失败**：显示失败原因，提供"重新支付"按钮
  - ⏳ **支付处理中**：显示轮询进度，提供"立即查询"按钮
  - ⚠️ **支付已取消**：提示用户，提供"重新支付"按钮

#### 技术实现

- 使用 Element Plus 的 `el-result` 组件展示不同状态
- 使用 `setInterval` 实现自动轮询
- 组件卸载时自动清除定时器，防止内存泄漏

### 4. 路由配置（`src/router/index.js`）

新增路由：

```javascript
{
  path: '/payment/result',
  name: 'PaymentResult',
  component: () => import('@/views/user/PaymentResult.vue'),
  meta: { title: '支付结果', requiresAuth: true }
}
```

### 5. 常量定义（`src/utils/constants.js`）

## 用户使用流程

### 前提条件

1. 用户已登录
2. 用户已创建订单（订单状态为"待支付"）

### 支付流程

1. **进入支付页面**
   - 方式一：在订单列表页点击"去支付"按钮
   - 方式二：在订单详情页点击"去支付"按钮
   - 方式三：直接访问 `/payment/{orderNo}`

2. **选择支付方式**
   - 默认选中"支付宝支付"
   - 也可选择"模拟支付"（测试环境）

3. **确认支付**
   - 点击"确认支付"按钮
   - 弹出确认对话框
   - 点击"确认支付"

4. **跳转到支付宝**
   - 页面自动跳转到支付宝收银台
   - 使用支付宝扫码或密码支付

5. **查看支付结果**
   - 支付完成后自动跳转到支付结果页
   - 显示支付状态和详细信息
   - 可点击"查看订单"跳转到订单详情

## 注意事项

1. **沙箱环境**
   - 当前使用支付宝沙箱环境
   - 需要使用支付宝沙箱买家账号进行支付
   - 生产环境需要更换为正式的支付宝应用配置

2. **安全性**
   - 所有支付接口都需要 JWT 认证
   - 支付单号由后端生成，防止伪造
   - 异步回调由后端处理，前端仅展示结果

3. **用户体验**
   - 支付过程中不要关闭浏览器窗口
   - 支付完成后请等待自动跳转
   - 如果支付状态长时间未更新，可手动刷新页面

4. **错误处理**
   - 网络错误会自动提示用户
   - 支付失败可以重新发起支付
   - 所有异常都会在控制台输出详细日志
